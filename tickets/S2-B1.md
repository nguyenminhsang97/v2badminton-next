# S2-B1: Wire Layout - Nav + Footer <- SiteSettings

## Objective

Cho `Nav` va `Footer` doc site name, phone, Zalo, Facebook tu Sanity thay vi `siteConfig`. Day la vertical slice nho nhat de verify pipeline Sanity -> layout -> props -> render.

## Input

- `getSiteSettings()` tu S2-A2
- `src/app/(site)/layout.tsx`
- `src/components/layout/Nav.tsx`
- `src/components/layout/Footer.tsx`

## Output

- `layout.tsx` fetch `getSiteSettings()`
- `Nav` nhan props thay vi import `siteConfig`
- `Footer` nhan props thay vi import `siteConfig`
- 2 component khong con runtime import tu `src/lib/site.ts`

Prop surface mong muon:

- `Nav`
  - `siteName`
- `Footer`
  - `siteName`
  - `phoneDisplay`
  - `phoneE164`
  - `zaloNumber`
  - `facebookUrl`

Co the pass them prop du phong neu muon, nhung khong can over-design.

## Dependency

- S2-A2

## Priority

- P0

## Acceptance Criteria

1. `Nav` render site name tu Sanity.
2. `Footer` render phone, Zalo, Facebook tu Sanity.
3. Edit `phoneDisplay` trong Studio -> sau revalidation -> `Footer` cap nhat.
4. Neu `getSiteSettings()` tra `null` -> `Nav` va `Footer` van render fallback hop ly.
5. `npm run build` pass.
6. `Nav.tsx` va `Footer.tsx` khong con import `siteConfig`.

## Edge Cases

### 1. `Nav` la client component

`Nav` dung `usePathname()` va `useEffect`, nen phai nhan data qua props. Khong fetch truc tiep trong component.

### 2. Layout-level caching

`(site)/layout.tsx` la server component. Fetch settings o day se duoc cache theo ISR strategy cua Sprint 2 va duoc share cho cac page trong route group. Day la behavior mong muon.

### 3. `routes.ts` chua doi trong ticket nay

`routes.ts` va metadata helpers van co the dung static config cho den S2-C2. Ticket nay chi xu ly layout UI, khong xu ly metadata.

### 4. Fallback object

Nen co 1 fallback object dua tren data cu de tranh `Footer` bi trong phone / social neu Sanity tam thoi unreachable.

## Non-Scope

- Chua them phone vao Nav visible
- Chua doi `routes.ts`
- Chua doi metadata generation
- Chua xoa `siteConfig`

## Risk / Luu y

- `Footer` la trust signal quan trong. Khong de blank phone number trong fallback path.
- B1 la `recommended order` de verify pipeline som, nhung khong phai hard dependency ky thuat cua B2.
