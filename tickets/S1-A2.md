# S1-A2: Setup Sanity Studio trong repo

## Objective

Cài Sanity Studio embedded trong v2badminton-next repo, chạy được local, kết nối đúng dataset production. Sau ticket này, dev có thể mở Studio trên localhost và thấy giao diện CMS trống sẵn sàng nhận schemas.

## Input

- Project ID từ S1-A1
- Dataset name: `production`
- Repo: `D:\V2\v2badminton-next`

## Output

- Sanity packages đã install trong project
- Sanity config file tạo xong (project ID, dataset, plugins)
- npm script `sanity:dev` chạy mở Studio trên localhost
- CORS origin cho localhost đã add trong Sanity dashboard
- `.env.local` chứa `NEXT_PUBLIC_SANITY_PROJECT_ID` và `NEXT_PUBLIC_SANITY_DATASET`
- `.env.example` cập nhật thêm 2 biến mới

## Dependency

- S1-A1 (cần project ID)

## Priority

P0

## Acceptance Criteria

1. `npm run sanity:dev` chạy không lỗi, mở trình duyệt thấy Sanity Studio
2. Studio kết nối đúng project và dataset `production`
3. Studio hiển thị empty state (chưa có schemas — đúng, sẽ thêm ở B1)
4. Env vars không bị commit vào git (check `.gitignore`)
5. Build Next.js vẫn pass (`npm run build` không break)

## Edge Cases

- Next.js 16 + Sanity: kiểm tra compatibility versions trước khi install. Đọc `node_modules/next/dist/docs/` nếu có breaking changes liên quan app router
- Nếu port conflict: Studio có thể chạy trên port khác, không bắt buộc port mặc định
- Windows path: đảm bảo sanity config không hardcode unix paths

## CMS Notes

- Studio nên được embed dưới dạng folder riêng trong repo (ví dụ `/sanity/` hoặc `/cms/`) hoặc theo Sanity v3 convention
- Không dùng Sanity Studio hosted (sanity.studio) — dùng embedded local để dễ customize sau
- Studio sẽ cần deploy lên Vercel hoặc host riêng sau Sprint 6 (cutover) — chưa cần lo bây giờ

## Non-Scope

- Chưa define schemas (S1-B1)
- Chưa cần API read token cho Next.js frontend (Sprint 2)
- Chưa cần deploy Studio lên production
- Chưa cần Sanity Vision plugin (nice-to-have, có thể thêm sau)

## Risk / Lưu ý

- AGENTS.md trong repo ghi: "Read the relevant guide in node_modules/next/dist/docs/ before writing any code" — dev phải check Next.js 16 docs trước khi thêm dependencies
- Sanity v3 là phiên bản hiện tại — không dùng v2
- Đừng install `@sanity/client` cho frontend ở ticket này — chỉ cần Studio packages
