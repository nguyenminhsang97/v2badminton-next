# S1-A1: Init Sanity Project

## Objective

Tạo Sanity project mới cho V2 Badminton, sẵn sàng để define schemas và nhập data. Sau ticket này, team có project ID, dataset, và ít nhất 1 account có quyền admin truy cập được.

## Input

- Tài khoản Sanity (đăng ký free tại sanity.io nếu chưa có)
- Tên project: `v2-badminton`
- Plan: Free (Growth free tier — đủ cho giai đoạn này)

## Output

- Sanity project tạo xong trên sanity.io/manage
- Project ID được ghi lại (sẽ dùng ở S1-A2)
- Dataset `production` đã tạo
- Owner account đã login và truy cập được dashboard
- Nếu có người khác cần truy cập (operator), invite với role **Editor** (không cần Admin — operator chỉ cần edit content, không cần manage project settings)

## Dependency

Không có.

## Priority

P0 — không có cái này thì không làm gì tiếp được.

## Acceptance Criteria

1. Truy cập sanity.io/manage → thấy project `v2-badminton`
2. Trong project → thấy dataset `production`
3. Owner login thành công, có full quyền
4. Ghi lại project ID vào `.env.local` hoặc doc nội bộ

## Edge Cases

- Nếu đã có Sanity account cũ: tạo project mới, không reuse project cũ
- Nếu tên `v2-badminton` trùng: dùng `v2-badminton-cms` hoặc tương tự
- Free plan giới hạn 3 non-admin users — đủ cho giai đoạn này, không cần upgrade

## Non-Scope

- Chưa cần setup Studio local (S1-A2)
- Chưa cần define schemas
- Chưa cần CORS settings (S1-A2 sẽ handle)
- Chưa cần API tokens cho Next.js (Sprint 2)

## Risk / Lưu ý

- Sanity free tier có giới hạn 100K API CDN requests/tháng — đủ cho dev + staging, nhưng cần monitor khi live
- Project ID là immutable sau khi tạo — chọn tên cẩn thận
- Không commit project ID hay token vào git — dùng env vars
