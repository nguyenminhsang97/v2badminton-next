# S2-A1: Sanity Client + Env Config

## Objective

Tao 1 module read-only duy nhat de frontend doc data tu Sanity production dataset. Module nay phai giai quyet 3 viec:

1. Client config: project ID, dataset, CDN, token, API version
2. Revalidation contract: xuat 1 constant dung chung cho Sprint 2
3. Null-safe fetch: khi Sanity null hoac loi mang thi frontend khong crash

## Acceptance Criteria

1. `import { sanityClient } from "@/lib/sanity/client"` build pass.
2. Client dung project `w58s0f53` va dataset `production`.
3. `useCdn: true`.
4. Khi co `SANITY_API_READ_TOKEN` trong env -> client dung token.
5. Khi khong co token -> client van hoat dong voi public dataset.
6. Export 1 revalidation constant ro rang, co comment giai thich ly do chon.
7. Null-safe wrapper nhan generic type `T` va fallback `T`, tra ve `T`, khong throw len caller.

   - `SANITY_API_READ_TOKEN=`
   - `NEXT_PUBLIC_SITE_URL=`
9. `npm run build` va `npm run lint` pass.
10. Ticket nay chi them module moi, khong break page/component hien tai.

## Edge Cases

### 1. `@sanity/client` co san hay chua

- Verify bang shell:
  - `node -e "require('@sanity/client')"`
- Neu import fail thi moi install rieng `@sanity/client`
- Khong assume package da co chi vi Studio dang chay

### 2. Revalidation strategy

Sprint 2 chon `ISR time-based`, khong dung webhook.

- Gia tri goi y: `60` giay
- Ly do:
  - don gian
  - khong can webhook endpoint
  - operator van thay update trong khoang chap nhan duoc
- Revalidation phai duoc export thanh 1 constant, khong hardcode rai rac

### 3. Null-safe wrapper

Phai handle ca 2 truong hop:

- Sanity tra `null`
- fetch/query bi network error, timeout, hoac rate limit

Pattern mong muon:

- singleton query -> fallback `null`
- array query -> fallback `[]`

### 4. Server-only

Client nay chi dung o server side:

- page.tsx
- layout.tsx
- server components
- query helpers

Khong import truc tiep vao `"use client"` components vi:

- token khong duoc expose
- fetch/revalidation la server concern
- client components phai nhan data qua props

Neu can, co the dung `server-only` de enforce.

### 5. `NEXT_PUBLIC_SITE_URL`

Bien nay khong dung de config Sanity client.

No duoc them tu Sprint 2 de:

- `robots.ts`
- `sitemap.ts`
- `canonicalUrl()`

khong phai doc `siteConfig.siteUrl` nua.
