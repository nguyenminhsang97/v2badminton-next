## Acceptance Criteria

1. Co the import tu `@/lib/sanity`.
2. Moi function return typed data, khong co `any`.
3. Tat ca queries mac dinh chi doc published documents.
4. `getFaqs("homepage")` tra dung `5` FAQs theo seed data hien tai.
5. `getScheduleBlocks()` resolve duoc `locationName`, khong null voi dataset production.
6. `getPricingTiers()` giu duoc union type cho `kind`.
7. Singleton query null-safe -> `null`; array query null-safe -> `[]`.
8. Moi query dung revalidation config tu S2-A1.
9. `npm run build` pass.

## Edge Cases

### 1. FAQ Portable Text

`answer` cua FAQ la Portable Text. Query can tra raw blocks, khong serialize thanh plain text o query level.

Type co the dung:

- `unknown[]`
- hoac `PortableTextBlock[]`

Ca 2 deu chap nhan duoc neu khong lam ro type gap Sprint 2.

### 2. Broken schedule references

Neu `schedule_block` reference toi location da bi xoa, dereference se tra `null`.

Recommend:

- filter `defined(location->_id)` o query level
- khong day broken data len UI

### 3. Singleton settings co the chua ton tai

Production da seed, nhung dataset moi co the chua co `site_settings`.

Function signature phai cho phep `null`, caller se handle fallback.

### 4. Published-only perspective

Vi dataset hien la public, query co the khong thay drafts.

Tuy vay, khi sau nay them token, client co the thay drafts. Nen uu tien 1 trong 2 cach:

- set `perspective: "published"` o client/fetch layer
- hoac them published-only filter nhat quan

Phai chot 1 cach de behavior khong doi khi env thay doi.

### 5. `locationId` va lead form contract

Query nen tra raw `locationId` de giu data trung thuc voi Sanity.

Nhung `ContactForm` va `submitLead` hien van validate theo legacy `CourtId`:

- `green`
- `hue_thien`
- `khang_sport`
- `phuc_loc`

Vi vay, raw `locationId` khong duoc dua thang vao form values ma khong qua adapter.
