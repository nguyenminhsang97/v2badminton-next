## Acceptance Criteria

1. Mo Studio -> Schedule Block -> thay 15 documents
2. Moi document link dung toi location (click reference -> mo dung location document)
3. `timeSlotId` khop 100% voi `timeSlotId` trong `schedule.ts`
4. `levels` array dung cho tung block
5. `isActive = true` cho tat ca
6. Tong 15 blocks, dung court distribution: Hue Thien 7, Green 4, Phuc Loc 3, Khang Sport 1

## Edge Cases

- Cung `timeSlotId` nhung khac `location` va `dayGroup` la hop le, khong phai duplicate
- Cung `location` + cung `timeSlotId` nhung khac `dayGroup` cung hop le
- `levelLabels` khong luu vao Sanity; frontend se derive tu `levels`
- Reference phai la Sanity document reference, khong luu string thuong

## CMS Notes

- Frontend se dung `timeSlotId` + `location` + `dayGroup` + `timeLabel` de build prefill message
- Khong luu `prefillMessage` trong Sanity
- Khi operator them lich moi, chi can tao document moi voi reference dung toi location

## Non-Scope

- Khong xoa `schedule.ts`
- Khong build prefill logic frontend
- Khong nhap data `doanh_nghiep` schedule vi source hien tai chua co

## Risk / Luu y

- 15 documents nhap tay kha nhieu, nen uu tien migration script neu can
- Neu script migrate dung field cu `timeKey` hoac `levelLabels`, import se fail hoac lech schema moi
