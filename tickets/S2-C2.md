# S2-C2: Refactor JSON-LD Builders <- Sanity Data

## Objective

Cho structured data va visible content dung cung 1 source of truth. Sau ticket nay, `schema.ts` khong con doc static content libs.

## Input

- `src/lib/schema.ts`
- `src/lib/routes.ts`
- `src/app/robots.ts`
- `src/app/sitemap.ts`
- Homepage va 3 existing pages da fetch Sanity data

## Output

`schema.ts` duoc refactor thanh bo pure builders nhan data params.

### Builder signatures mong muon

- `buildOrganizationSchema({ siteUrl, siteName, phoneE164, facebookUrl })`
- `buildWebsiteSchema({ siteUrl, siteName, language })`
- `buildFaqPageSchema(faqs)`
- `buildHomepageLocalBusinessSchema({ siteUrl, settings, locations, priceRange })`
- `buildLocalPageBusinessSchema({ siteUrl, path, settings, locations, priceRange })`
- `buildCourseSchemas({ siteUrl, siteName, pricingTiers })`

## Dependency

- S2-B2
- S2-C1

## Priority

- P0

## Acceptance Criteria

1. `schema.ts` co zero imports runtime tu:
   - `@/lib/faqs`
   - `@/lib/pricing`
   - `@/lib/locations`
   - `@/lib/site`
2. Homepage `FAQPage` schema questions khop visible FAQ content.
3. Homepage `LocalBusiness` phone khop Footer phone.
4. 3 existing pages co JSON-LD khop visible content sau khi wire.
5. `robots.ts` khong con import `siteConfig`.
6. `robots.ts` va `sitemap.ts` dung `NEXT_PUBLIC_SITE_URL` lam canonical base.
7. `npm run build` pass.

## Edge Cases

### 1. Portable Text -> plain text cho FAQ schema

`FAQPage` schema can `acceptedAnswer.text` la plain text string. Can co helper chuyen Portable Text blocks thanh chuoi plain text.

Khong can them dependency moi neu utility tu viet du.

### 2. `siteUrl` la env, khong phai Sanity

`siteUrl` khong nam trong `site_settings` schema. Nen canonical base phai dung:

- `process.env.NEXT_PUBLIC_SITE_URL`

Khong fetch Sanity cho viec nay.

### 3. Language / locale khong nam trong CMS

`site_settings` hien khong co `language` hoac `locale`. `buildWebsiteSchema()` va `buildMetadata()` co the nhan:

- static constant
- hoac default `"vi-VN"`

Day la config cap app, khong phai CMS content.

### 4. `routes.ts` partial refactor la du

`coreRoutes` van la static config va co the giu trong code. Ticket nay khong can bien route titles/descriptions thanh CMS data. Dieu can doi la:

- canonical base URL
- cac builder khong con phu thuoc vao static content libs

### 5. Price range cho LocalBusiness

`sitePriceRange` hien la static derived value. Sau khi wire Sanity, phai derive lai tu `SanityPricingTier[]` hoac pass san vao builder. Khong import lai static helper.

### 6. Local page descriptions

Cac mo ta SEO hardcoded trong `buildLocalPageBusinessSchema()` co the giu tam o Sprint 2. Day khong phai content source-of-truth issue vi no chua ton tai trong CMS.

## Non-Scope

- Khong doi `coreRoutes` thanh CMS data
- Khong can dynamic sitemap tu money pages moi
- Khong quan ly metadata bang Sanity

## Risk / Luu y

- Neu plain text conversion cho FAQ sai, structured data se lech visible content.
- Sau refactor, `schema.ts` nen de test hon vi tro thanh pure functions that su.
