# S1-B2: Schemas cho trust + conversion data

## Objective

Define 2 Sanity schemas cho data chưa có sẵn nhưng cần model trước: coach và testimonial. Sau ticket này, Studio sẵn sàng nhập khi owner cung cấp ảnh HLV và testimonial thật.

## Input

- Content models v2 spec (đã chốt)

## Output

2 schemas live trong Sanity Studio:

### 1. `coach`
| Field | Type | Required | Notes |
|---|---|---|---|
| name | string | yes | Tên thật |
| photo | image | no | Ảnh chân dung |
| photoAlt | string | no | Alt text cho ảnh |
| teachingGroup | string | yes | VD: Lớp cơ bản cho người mới |
| approach | text | yes | 1 câu mô tả phương pháp dạy |
| order | number | no | Thứ tự hiển thị |
| isActive | boolean | yes | Default: true |

### 2. `testimonial`
| Field | Type | Required | Notes |
|---|---|---|---|
| studentName | string | yes | Tên hoặc tên viết tắt |
| studentGroup | string, list | yes | Options: `tre_em`, `nguoi_moi`, `nguoi_di_lam`, `doanh_nghiep` |
| contextLabel | string | no | VD: Phụ huynh học viên lớp hè, Nhân viên văn phòng |
| content | text | yes | Nội dung review |
| order | number | no | |
| isActive | boolean | yes | Default: true |

## Dependency

- S1-A2 (Studio phải chạy được)
- Không cần B1 — hai schemas này không reference tới schemas khác

## Priority

P1 — schemas cần có sớm để model sẵn, nhưng data sẽ nhập sau (Sprint 3)

## Acceptance Criteria

1. Mở Studio → thấy Coach và Testimonial trong sidebar
2. Tạo thử 1 coach document → form hiển thị đúng fields
3. Tạo thử 1 testimonial → studentGroup dropdown có 4 options
4. Xóa test documents sau khi verify

## Edge Cases

- Coach không có ảnh: photo là optional — Studio không nên block save
- Testimonial rất ngắn (1 câu): chấp nhận, không đặt min length

## Non-Scope

- Chưa nhập data thật (cần owner cung cấp ảnh + nội dung → Sprint 3)
- Chưa kết nối frontend
- Chưa cần coach linked to schedule/location (có thể thêm sau nếu cần)

## Risk / Lưu ý

- Coach schema cố ý đơn giản: tên + nhóm dạy + approach. Không có fields "thành tích thi đấu" hay "chứng chỉ" — đây là quyết định product có chủ đích (xem Blueprint)
- Testimonial nên dùng nội dung thật 100% — không fake, không inflate
