# S2-B4: Wire ContactForm <- Sanity Data

## Objective

Cho `ContactForm` nhan locations, time slots, va Zalo data tu Sanity thay vi static imports. Day la component lon nhat trong Sprint 2, nen ticket nay chi doi data source, khong doi business logic.

## Input

- `src/components/home/ContactForm.tsx` (hien tai la component lon 600+ lines)
- `src/components/home/ContactSection.tsx`
- Query data da duoc fetch o `page.tsx` sau S2-B2
- Validation/server action hien tai van dua vao legacy enums trong:
  - `src/lib/validation/lead.ts`
  - `src/lib/locations.ts`
  - `src/lib/schedule.ts`

Static imports can thay:

- `courtLocations`
- `scheduleItems`
- `siteConfig`

## Output

`ContactForm` nhan props:

- `locations: SanityLocation[]`
- `timeSlots: { value: string; label: string }[]`
- `zaloNumber: string`

`ContactSection` nhan data tu `page.tsx` va pass vao `ContactForm`.

## Dependency

- S2-B2

## Priority

- P0

## Acceptance Criteria

1. Court dropdown populate dung 4 courts tu Sanity.
2. Time slot dropdown populate dung unique time slots.
3. Zalo link dung.
4. Form submit van hoat dong end-to-end.
5. Schedule card click -> form prefill dung court + time slot + message.
6. Business mode van compose dung message tu van doanh nghiep.
7. `ContactForm.tsx` khong con import `courtLocations`, `scheduleItems`, `siteConfig`.
8. `npm run build` pass.

## Edge Cases

### 1. Court dropdown values va validation contract

Day la review point quan trong nhat.

`submitLead` va `validateLeadFieldEnums` hien van whitelist `court` theo legacy `CourtId`:

- `green`
- `hue_thien`
- `khang_sport`
- `phuc_loc`

Do do:

- khong duoc dua raw Sanity `_id` nhu `location.green` vao form field `court`
- phai normalize sang legacy `CourtId`

Neu dev muon doi dropdown value sang raw Sanity id, thi validation/server action khong con la non-scope nua.

### 2. Time slot values

`time_slot` hien van duoc validate theo `TimeSlotId` trong static file. May man la Sanity dang dung cung machine keys, nen dropdown values nen giu nguyen `timeSlotId`.

### 3. Time slot options nen pre-compute

Unique time slots nen duoc derive tu `SanityScheduleBlock[]` o page hoac `ContactSection`, roi pass vao form duoi dang:

- `{ value, label }[]`

`ContactForm` khong can biet full schedule blocks.

### 4. Type imports

Hien `ContactForm` import `ScheduleLevel` tu static schedule file. Neu co the, chuyen type nay sang local union hoac re-export tu `@/lib/sanity` de giam dependency.

## Non-Scope

- Khong doi submit logic
- Khong doi validation rules
- Khong doi Turnstile
- Khong doi tracking
- Khong doi `HomepageConversionProvider`

## Risk / Luu y

- Day la component lon, de vo regression neu refactor qua tay.
- Giu thay doi o muc data source va option mapping, dung sua logic ngoai scope.
