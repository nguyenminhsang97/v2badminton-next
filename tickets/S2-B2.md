# S2-B2: Wire Homepage Data - Fetch + Distribute Props

## Objective

Bien homepage `page.tsx` thanh data orchestrator: fetch tat ca Sanity data can thiet roi pass xuong sections qua props. Ticket nay lo phan plumbing, chua doi internal rendering logic cua tung section.

## Input

- `src/app/(site)/page.tsx`
- Query functions tu S2-A2
- Pattern server fetch -> client props da duoc verify o S2-B1

## Output

`page.tsx` goi:

- `getPricingTiers()`
- `getScheduleBlocks()`
- `getLocations()`
- `getFaqs("homepage")`
- `getSiteSettings()`

Va pass data xuong:

- `PricingSection`
- `ScheduleSection`
- `LocationsSection`
- `FaqSection`
- `ContactSection`

Moi section can co props interface ro rang.

## Dependency

- S2-A2

## Priority

- P0

## Acceptance Criteria

1. Homepage `page.tsx` tro thanh `async` server component.
2. Data flow la: `page.tsx` fetch -> props -> section render.
3. Moi section co props interface typed ro rang.
4. Client components nhan data qua props, khong tu fetch.
5. `npm run build` pass.
6. Homepage van render day du sections voi visual parity co ban.
7. `page.tsx` khong con import static content libs.

## Edge Cases

### 1. Provider boundary

Homepage hien duoc wrap boi `HomepageConversionProvider`, la client component. Data phai di qua boundary nay duoi dang plain objects serializable.

Pattern hop le:

```text
page.tsx (server, fetch)
  -> HomepageConversionProvider (client)
    -> PricingSection / ScheduleSection / ContactSection (client)
```

### 2. JSON-LD chua doi

Homepage hien van render `JsonLd` tu `schema.ts` va cac builder van doc static data. Ticket nay chua xu ly phan do. C2 moi la ticket dong bo source of truth cho structured data.

### 3. `ContactSection` chi nhan props, chua doi internals

Ticket nay co the phai them props interface cho `ContactSection`, nhung wiring chi tiet vao `ContactForm` la S2-B4.

### 4. Type compatibility

`PricingSection` va `ScheduleSection` la client components, nen prop types phai gon, serializable, va khong dua functions qua props.

## Non-Scope

- Chua doi internal logic cua sections
- Chua doi `ContactForm`
- Chua refactor JSON-LD
- Chua doi UI/CSS

## Risk / Luu y

- Ticket nay la phan plumbing. Neu vuot scope sang refactor internals se kho review hon va kho isolate regression.
