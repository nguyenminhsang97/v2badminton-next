# S1-D1: Operator Acceptance Test

## Objective

Confirm rang mot nguoi khong biet code van co the login vao Sanity Studio, tim dung content, chinh sua, luu, va quay lai trang thai ban dau ma khong can dev can thiep.

## Input

- Sanity Studio da chay duoc local tai `/studio`
- Toan bo data Sprint 1 da migrate xong:
  - 1 `site_settings`
  - 4 `location`
  - 5 `pricing_tier`
  - 15 `schedule_block`
  - 21 `faq`
- Operator co tai khoan Sanity de dang nhap

## Output

Operator tu thuc hien 4 test cases:

### Test 1: Edit gia hoc phi

- Mo Studio -> Pricing Tier -> chon `3 buoi / tuan`
- Doi `Price Per Month` tu `1300000` sang `1350000`
- Save / Publish
- Verify gia moi da duoc luu
- Doi lai ve `1300000`, save / publish

### Test 2: Sua noi dung FAQ

- Mo Studio -> FAQ -> chon FAQ `Toi chua bao gio choi cau long, co hoc duoc khong?`
- Them ` (test)` vao cuoi answer
- Save / Publish
- Verify thay doi da duoc luu
- Xoa ` (test)`, save / publish lai

### Test 3: Tao location test roi xoa

- Mo Studio -> Location -> tao moi 1 document
- Nhap toi thieu cac field sau:
  - `Name = San Test`
  - `Slug = san-test`
  - `Short Name = Test`
  - `District = Binh Thanh`
  - `Address Text = 123 Test Street, Binh Thanh, Ho Chi Minh`
  - `Maps URL = https://example.com/test-location`
  - `Is Active = false`
- Save / Publish
- Verify document tao thanh cong
- Xoa document test

### Test 4: Verify reference navigation

- Mo Studio -> Schedule Block -> chon bat ky block nao
- Click vao `Location` reference
- Verify Studio mo dung location document
- Quay lai

## Dependency

- S1-C1
- S1-C2
- S1-C3
- S1-C4

## Priority

P0

## Acceptance Criteria

1. Operator tu login duoc vao Studio.
2. Operator tu hoan thanh 4 test cases tren.
3. Operator xac nhan rang ho hieu cach navigate Studio va co the tu sua content.
4. Khong co document that bi sua sai sau khi test.
5. Document test duoc xoa, data cuoi cung tro ve trang thai baseline.

## Non-Scope

- Khong test frontend rendering
- Khong test campaign, money page, post editing
- Khong test mobile UI cho Studio
- Khong test publish workflow phuc tap hoac roles nang cao

## Risk / Luu y

- Ticket nay can nguoi that thao tac trong Studio, dev khong the tu pass hoan toan bang CLI.
- Neu muon strict operator acceptance, nen dung mot account tach biet voi owner/dev.
- Test tao location se fail neu bo sot `shortName` hoac `mapsUrl`, vi day la cac field required trong schema.
