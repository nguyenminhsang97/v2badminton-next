## Component Notes

### 1. `FaqSection`

- Nhan `SanityFaq[]`
- Render `answerBlocks` bang Portable Text
- Install `@portabletext/react` neu chua co
- Neu answer rong -> render nothing, khong crash

### 2. `PricingSection`

- Nhan `SanityPricingTier[]`
- Map dung vao 3 card variants:
  - group
  - private
  - enterprise
- Khong import `sitePriceRange` tu static file

  - compute local tu Sanity tiers
  - hoac nhan qua props
- Khong duoc keo lai static dependency

### 3. `ScheduleSection`

- Nhan `SanityScheduleBlock[]`
- Khong dung `courtLocationMap` hoac `scheduleItems`
- Card display dung:
  - `locationShortName`
  - `dayGroup`
  - `timeLabel`
  - `levels`
- Prefill message compose tu Sanity fields:
  - `Quan tam lich: {dayGroup} | {timeLabel} | {locationName}`

Review note cho tabs:

- Prefer derive tabs tu unique locations trong schedule data
- Giu cung thu tu hien tai neu co the:
  - `all`
  - `hue_thien`
  - `green`
  - `phuc_loc`
  - `khang_sport`
- Neu khong derive, phai co adapter ro rang giua Sanity location data va tab ids hien tai

### 4. `LocationsSection`

- Nhan `SanityLocation[]`
- Render:
  - ten san
  - district label tieng Viet
  - dia chi
  - link ban do
- `district` trong Sanity la machine key, can map ra label:
  - `binh_thanh` -> `Binh Thanh`
  - `thu_duc` -> `Thu Duc`

## Acceptance Criteria

1. Homepage render dung data tu Sanity, khong trong section nao.
2. FAQ answers render rich text dung, khong chi plain text.
3. Pricing cards render dung variant theo `kind`.
4. Schedule tabs loc dung theo san.
5. Click schedule card -> prefill message compose dung.
6. Location cards hien district label tieng Viet, khong hien raw machine key.
7. 4 files khong con import static runtime content libs.
8. `npm run build` va `npm run lint` pass.

## Edge Cases

### 1. Portable Text answer

Nen render theo pattern:

- `<PortableText value={faq.answerBlocks} />`

Khong dung `dangerouslySetInnerHTML` nua cho data FAQ tu Sanity.

### 2. `kind` type narrowing

`SanityPricingTier.kind` phai la union type. Neu query layer tra `string`, `switch (tier.kind)` se mat type safety.

### 3. Schedule tab IDs va form prefill

`ScheduleSection` co the lam viec voi raw `locationId` cho display/filter, nhung neu prefill lien quan den lead form thi phai normalize sang legacy `CourtId`.

### 4. Price range text

Static helper `sitePriceRange` khong ton tai trong Sanity. Neu UI con can hien chuoi nay thi phai derive tu data Sanity, khong duoc import lai static file.
